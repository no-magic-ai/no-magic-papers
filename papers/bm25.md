## TL;DR

This monograph presents the Probabilistic Relevance Framework — the theoretical foundation behind the BM25 retrieval scoring function — and shows how the BM25 formula's term-frequency saturation, length normalization, and IDF weighting follow naturally from a probability-of-relevance derivation rather than from ad-hoc tuning.

## Problem

Information retrieval before the Probabilistic Relevance Framework relied on TF-IDF heuristics: term frequency rewarded matches, IDF down-weighted common words, and document length was handled crudely or ignored. The heuristics worked but did not say which terms should saturate, how length normalization should interact with frequency, or how to derive better scoring functions from first principles. BM25 emerged from the Okapi system in TREC-3 (Robertson et al. 1995) as a particular formula that happened to work well; this 2009 monograph gives the rigorous probabilistic-relevance derivation, the BM25 family of variants (BM25F for fielded documents, BM25-extensions for query terms with weights), and the empirical regimes where each variant applies.

## Contribution

The monograph derives BM25 from a probabilistic model of relevance: assume documents and queries are bags of terms drawn from class-conditional distributions (relevant vs non-relevant), invoke binary independence and a few empirical adjustments (eliteness for term frequency saturation, length normalization to avoid penalizing long documents proportionally), and the resulting log-odds score is BM25. The monograph then catalogs the family. Plain BM25 scores a document d for a query q as Σ over query terms t of IDF(t) · (tf(t,d) · (k1+1)) / (tf(t,d) + k1 · (1 - b + b · |d|/avgdl)), with two free parameters k1 (term-frequency saturation) and b (length normalization). BM25F generalizes to documents with multiple fields (title, body, anchor text) by per-field length normalization before the per-document combination. The framework also yields BM25 with relevance feedback, BM25 over query expansion, and connections to language-modeling retrieval.

## Method summary

- For each query term t, compute IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5)), where N is corpus size and df(t) is document frequency.
- For each document d and term t, compute the saturated term-frequency term: TF_norm(t,d) = (tf(t,d) · (k1+1)) / (tf(t,d) + k1 · (1 - b + b · |d|/avgdl)).
- Document score: score(q,d) = Σ_t IDF(t) · TF_norm(t,d), summed over query terms.
- Standard parameter values: k1 ∈ [1.2, 2.0], b ≈ 0.75 — the monograph documents the empirical justification and the regimes where alternative values help.
- BM25F extension: per-field length normalization before combining; per-field weights are then learned or hand-set.
- Implementation pattern: build an inverted index of (term → posting list of (doc_id, tf)); at query time, walk posting lists for query terms and accumulate per-document scores using the formula above.

## Key results

The monograph synthesizes two decades of TREC results showing BM25 outperforms plain TF-IDF and pure language-model retrieval on most benchmarks at typical parameter settings, while remaining cheap to compute, easy to index, and interpretable. BM25F outperforms plain BM25 on web-style retrieval where field structure is meaningful (title vs body). The framework also explains why some plausible-looking variants underperform: ad-hoc tuning that violates the probabilistic-relevance assumptions tends to lose generality across collections.

## Relation to existing work

BM25 traces operationally to the Okapi system at City University London (Robertson et al. 1994, 1995 TREC papers) and theoretically to the binary-independence model (Robertson & Sparck Jones 1976). The Probabilistic Relevance Framework unifies these threads. BM25 contrasts with language-model retrieval (Ponte & Croft 1998), which scores by P(query | document) using a smoothed unigram model; the two families converge in some limits and the monograph discusses the relationship. Modern dense-retrieval methods (DPR, ColBERT, embedding-based search) supersede BM25 in many neural pipelines, but BM25 remains the strongest sparse-retrieval baseline and is the default first stage in most hybrid retrieval systems (BM25 + reranker, BM25 + dense retrieval ensemble). RAG-style systems often use BM25 alongside dense retrieval to capture exact-match signals dense models miss.

## Implementation notes

A pedagogical script can implement BM25 from scratch on a small corpus. Minimum viable implementation: tokenize and lowercase, build an inverted index (term → list of (doc_id, term_frequency)), compute document lengths and average length, score queries by walking posting lists. Pitfalls: omitting the +0.5 smoothing in IDF; failing to use avgdl correctly; confusing query-term frequency with document-term frequency. Useful exercise: progress from raw TF to TF-IDF to full BM25 in one script and observe rank-stability differences.

## Open questions

BM25's parameters (k1, b) are typically set by global heuristic; per-collection tuning improves results but the optimal values depend on collection characteristics. The integration with neural retrievers is an active practical question.
